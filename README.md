An end-to-end AI system that analyzes customer reviews, classifies sentiment, and detects recurring complaint categories using NLP and Machine Learning.

| File | Description |

## How to Run the Notebook on Google Colab

1. Go to [colab.research.google.com](https://colab.research.google.com)
2. Open the notebook: **File → Upload notebook** → select `AI_Customer_Feedback_Intelligence_System.ipynb`
3. Upload the dataset: click the **Files** icon on the left sidebar → **Upload** → select `Womens Clothing E-Commerce Reviews.csv`
   - Make sure it's uploaded into the root session folder (`/content/`), matching the path used in the notebook's `pd.read_csv(...)` cell.
4. Add this line in a new cell **at the very top** of the notebook (Colab doesn't have `wordcloud` pre-installed):
   ```python
   !pip install wordcloud -q
   ```
5. Download NLTK data by adding this to the same first cell (or right after the imports cell):
   ```python
   import nltk
   nltk.download('stopwords')
   nltk.download('punkt')
   ```
6. Run everything: **Runtime → Run all**

Expected total runtime: 2-5 minutes.

> **Note:** Colab sessions are temporary — files you upload (and files the notebook generates, like `cleaned_customer_reviews.csv` or the `.pkl` files) will disappear when the session ends. Download them from the Files panel (right-click → Download) before closing Colab if you want to keep them.

## Bonus Extras

### 1. Interactive Dashboard (`dashboard.html`)
A self-contained results dashboard — just double-click the file to open it in your browser (Chrome, Edge, etc). No installation, no server needed. It has 5 tabs: Overview, Sentiment Analysis, Model Comparison (with a toggle for Accuracy/F1/Precision), Complaint Clusters, and Business Insights.

### 2. Deployment Demo — CLI (`predict.py`)
Loads the saved model and classifies a review you type in:
```bash
pip install nltk scikit-learn
python predict.py "This dress is gorgeous and fits perfectly!"
```
Run it with no arguments to see it classify 4 built-in example reviews instead.


Both `predict.py` and `app.py` need `sentiment_model.pkl` and `tfidf_vectorizer.pkl` in the same folder — copy them alongside these scripts before running.

## Troubleshooting

| Issue | Fix |
|---|---|
| `FileNotFoundError` when reading the CSV | Re-upload the dataset file — Colab clears uploaded files when the session restarts/disconnects |
| `LookupError: stopwords not found` | Run the NLTK download cell (step 5) again |
| `ModuleNotFoundError: No module named 'wordcloud'` | Run `!pip install wordcloud -q` (step 4) |
| Runtime disconnected / session expired | Runtime → Reconnect, then re-upload the dataset and run all cells again |
| Code is slow | Reduce `max_features` in TF-IDF, or `n_estimators` in Random Forest |
